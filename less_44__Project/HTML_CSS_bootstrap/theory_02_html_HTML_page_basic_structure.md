# 1. Базовая структура HTML-страницы

## Краткое описание

Перед вами готовый HTML-шаблон с уже подключённым Bootstrap.

Bootstrap — это библиотека стилей, которая помогает быстрее создавать красивые и адаптивные сайты.

В этом шаблоне:

* уже подключены стили Bootstrap;
* уже подключён JavaScript Bootstrap;
* основная задача — изменять содержимое только внутри области страницы.

---

## Полный шаблон страницы

```html
<!doctype html>
<html lang="en">
    <head>
        <meta charset="utf-8">

        <meta
            name="viewport"
            content="width=device-width, initial-scale=1"
        >

        <title>Bootstrap demo</title>

        <link
            href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css"
            rel="stylesheet"
        >
    </head>

    <body>
        <!-- Начало страницы -->

        <h1>Hello, world!</h1>

        <!-- Конец страницы -->

        <script
            src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.bundle.min.js">
        </script>
    </body>
</html>
```

---

# Разбор структуры

## `<!doctype html>`

Сообщает браузеру:

> «Документ написан на HTML5».

```html
<!doctype html>
```

---

## `<html lang="en">`

Корневой контейнер страницы.

Атрибут `lang="en"` указывает язык страницы.

```html
<html lang="en">
```

Полезный, но не обязательный атрибут. Нужен для:
* Скринридеры и доступность
  * Программы для озвучивания текста выбирают правильное произношение и голос.
* Перевод страниц
  * Браузер понимает, нужен ли автоматический перевод.
* SEO
  * Поисковики лучше определяют язык страницы и показывают её нужной аудитории.
* Проверка орфографии
  * Браузер выбирает словарь для spellcheck.
* CSS и типографика
  * Некоторые стили и правила могут зависеть от языка, например выбора шрифта

---

## `<head>`

Служебная часть страницы.

Здесь обычно находятся:

* настройки;
* подключение стилей;
* метаинформация.

```html
<head>
    ...
</head>
```

---

## `<meta charset="utf-8">`

Указывает кодировку страницы.

UTF-8 позволяет корректно отображать:

* на ascii-символы;
  * (иначе вместо кириллицы вылезут какие-то краказябры)
* английский текст;
* специальные символы.



```html
<meta charset="utf-8">
```

---

## `<meta name="viewport">`

Позволяет странице правильно отображаться на телефонах и планшетах.

```html
<meta
    name="viewport"
    content="width=device-width, initial-scale=1"
>
```

---

## `<title>`

Название вкладки браузера.

```html
<title>Bootstrap demo</title>
```

---

## Подключение Bootstrap

Bootstrap подключается через тег `<link>`.

```html
<link
    href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css"
    rel="stylesheet"
>
```

После подключения становятся доступны:

* готовые стили;
* сетка Bootstrap;
* кнопки;
* карточки;
* формы;
* компоненты интерфейса.

---

## `<body>`

Основное содержимое страницы.

Именно здесь размещается:

* текст;
* изображения;
* кнопки;
* таблицы;
* формы.

```html
<body>
    ...
</body>
```

---

# Рабочая область страницы

## Важное правило

Менять код можно только между комментариями:

```html
<!-- Начало страницы -->

<!-- Конец страницы -->
```

Именно здесь размещается весь пользовательский контент.

---

## Пример

```html
<!-- Начало страницы -->

<h1>Hello, world!</h1>

<p>
    Это мой первый сайт на Bootstrap.
</p>

<!-- Конец страницы -->
```

---

# Подключение JavaScript Bootstrap

В конце страницы подключается JavaScript Bootstrap.

```html
<script
    src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.bundle.min.js">
</script>
```

Он нужен для работы:

* модальных окон;
* выпадающих меню;
* каруселей;
* интерактивных компонентов Bootstrap.
